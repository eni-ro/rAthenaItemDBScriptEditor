# 追加要件

## 目的
1. DBの指定をファイル直接編集ではなくアプリ上で指定できるようにする
2. アイテムが持つスクリプトだけでなく、セットボーナス用のスクリプトも管理できるようにする
3. アイテムの検索名をユーザの第一言語で検索できるようにする

## 実装
### スクリプト編集画面の分離
既存のメイン画面をScript編集画面(モーダルダイアログ)としてメイン画面から分離する
既存の画面はアイテムのスクリプトをロード、セーブする機能を持っていたが、メイン画面から渡されたスクリプトを編集し、メイン画面に返すだけの機能にする

### メイン画面の変更
#### 設定メニュー
メイン画面は、メニューバーでSettingsをクリックするとSettings画面に遷移できるようにする  
Settings画面では既存のdb.ymlの中のItem、Mob、Skillのパスに加えてItemCombos、ItemNameのパスも追加編集できるようにする  
設定は従来同様に、exeファイルと同じフォルダに置いてあるdb.ymlに書き込むこと

ItemCombosのフォーマットは以下の例のようになっている
これは同じコンボ効果を持つ複数のアイテムの組み合わせを表現できるフォーマットになっている
```yaml
Header:
  Type: COMBO_DB
  Version: 1

Body:
  - Combos:
      - Combo:
          - ItemAegisName1
          - ItemAegisName2
      - Combo:
          - ItemAegisName3
          - ItemAegisName4
    Script: |
      bonus2 bAddRace,RC_Dragon,5;
```

ItemNameのフォーマットは以下のようになっている
Itemで指定されたDBから構築したItemのリストに対して、同一IDのアイテムに対して別の検索名を付与することで、ユーザの第一言語でアイテムの検索ができるようにする


```yaml

Header:
  Type: ITEM_JPNAME_DB
  Version: 1

Body:
  - Id: 501
    Name: 赤ポーション
  - Id: 502
    Name: 紅ポーション
  - Id: 503
    Name: 黄ポーション
  - Id: 504
    Name: 白ポーション
  - Id: 505
    Name: 青ポーション
  - Id: 506
    Name: 緑ポーション
  - Id: 507
    Name: 赤ハーブ
  - Id: 508
    Name: 黄ハーブ
```  

#### メイン画面
メイン画面はアイテム検索画面と、タブから成る  
タブではアイテム編集モードと、アイテムセットボーナス編集モードを切り替えできるようにする  

アイテム検索画面ではアイテムを検索すると、AegisName、Name、検索名のいずれかで部分一致したアイテムDBのアイテムそのものと、コンボ効果のItemComboのリストが表示される
そこでアイテムそのものをクリックするとアイテム編集モードが、コンボをクリックするとコンボ編集モードが表示される
アイテム名の表示は、検索名が存在する場合は検索名(AegisName),存在しない場合はName(AegisName)のように表示すること。
リストにはアイテムDBのアイテムを表示する場合はItem:アイテム名と表示すること。
コンボを表示する場合は当該Combo:Comboアイテム名を-で連結して表示すること。ここで表示するのは当該Comboに含まれるアイテムのセットのみであり、Combosブロックに含まれるすべてのComboを表示するのではない。

##### アイテム編集モード
アイテム編集モードでは、既存の機能の他に、item_db_equip.yml先頭に記載のフォーマットに従ったアイテムデータを全て編集できるようにする  
各要素の設定は独立した入力欄を設ける
Jobs,Classes,Locationsは複数選択できるリストになる。選択可能な値は`sample/doc/item_db.txt`の内容から項目を作ってほしい。
リストで選択されたアイテムは、DBへの書き込み時には`Crusader: true`のようにキー名:trueの形式で出力すること。リストにないものは出力しないこと。ただし、リストから一つも選択されていない場合、TypeがWeaponまたはArmorの場合は`All: true`を出力してほしい。WeaponかArmorでない場合は何も出力しないこと。

また、項目によってはデフォルト値を持っているため、欄が空欄の場合は入力部分をグレーアウトして、デフォルト値で動くことを示してほしい
スクリプト部分は直接編集のテキストボックスを用意しておいて、横にあるボタンからScript編集画面を起動して編集もできるようにする

##### コンボ編集モード
コンボ編集モードでは、当該Combosにぶら下がるコンボ一つ一つの追加編集削除を既存のアイテム検索画面を利用して行えるようにする。
アイテム検索画面は既存のアイテム検索用ダイアログを流用して利用する。
また、AegisNameだけの表示ではどのアイテムとの組み合わせなのか視認性が悪いため、検索名か、検索名が存在しない場合はNameを表示すること。
スクリプト部分は直接編集のテキストボックスを用意しておいて、横にあるボタンからScript編集画面を起動して編集もできるようにする

### YAML操作バックエンド（Python EXE）の導入

#### 1. 導入の目的
手作業で編集された不規則なYAMLファイルにおいて、既存のコメント、改行、項目の記載順序を100%維持したまま、特定のデータのみを正確に更新・削除するため、`ruamel.yaml` ライブラリを使用したPython製バックエンドを導入する。

#### 2. システム構成
- **技術スタック**: Python 3.10+ / `ruamel.yaml`
- **配布形態**: `PyInstaller` 等を使用して単一の実行ファイル（例: `db_processor.exe`）にビルドし、Tauriの **Sidecar** 機能としてアプリに同梱する。
- **通信方式**: Tauri（Rust）から、書き込みが必要なタイミングで子プロセスとしてワンショット起動する。
- **実行環境**: ユーザ環境のPython有無に依存しないよう、EXE化したものを同梱すること。

#### 3. 処理仕様
1. **動作フロー**: 
    1. 保存実行時に、Rust側から必要なデータ（ファイルパス、対象ID、更新データ等）をコマンドライン引数または標準入力経由でJSON形式で渡す。
    2. Python EXEが起動し、対象YAMLを `ruamel.yaml` (RoundTripMode) でロードする。設定は`indent(mapping=2, sequence=4, offset=2)`とする。
    3. 特定のアイテムまたはコンボのブロックを検索し、指示されたフィールドのみを書き換える。
    4. ファイルに上書き保存し、成否を標準出力経由でRust側に返して終了する。
2. **安全性**:
    - 書き込み前に自動的にバックアップ（`.bak`）を作成すること。
    - 編集箇所以外のコメントや改行位置が一切変更されないことを最優先とする。
