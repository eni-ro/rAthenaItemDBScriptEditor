# 概要
開発対象のツールは、Ragnarok Onlineのエミュレータサーバであるrathenaで使用されているアイテムDBの定義ファイル(YAML形式)のエディタである  
rathenaのアイテムのスクリプト記述では、あらかじめ用意されているスクリプト用関数(ここではビルドインスクリプトと呼ぶ)を組み合わせることでアイテム全体のスクリプト(ここではアイテムスクリプトと呼ぶ)の挙動を定義することが可能であるが、所望のビルドインスクリプトをマニュアルを読むことなく簡単に入力することが目的となる  

# 技術スタック
アプリはtauri(Typescript+Vue+Rust)で開発する  
Vueの表示用コンポーネントとしてはVuetifyを優先的に採用する  
ただし、エディタ部分はオートコンプリート機能の拡充を考慮してMonaco Editorを採用する  
もしVuetiryとMonaco Editorでコンポーネントの機能が不足する場合は、候補となるものを提案すること  

# UI
メイン画面はsample\image\main.pngと同じコンポーネント配置とする  
左半分がアイテムスクリプトに入力するビルドインスクリプトの設定画面で、右半分がアイテムスクリプトの編集画面となる  

## アイテムスクリプトの表示
右上のLoadを押すと、sample\image\item_select.pngのような画面が表示される画面で編集するアイテムを選択する  
アイテム一覧はアプリ起動時にsample\input\db.ymlで指定された複数のYAMLを統合して表示される  
例えばsample\input\item_db_equip.ymlのようなファイルが登録されている  
このYAMLファイルのNameとAegisNameがアイテム選択画面で表示される  
選択画面ではNameまたはAegisNameによるフィルタリングができること  
例えば文字を入力すると、sample\image\item_select filtered.pngのようにフィルタリングされる
アイテムを選択すると、sample\image\script loaded.pngのように右画面にScript、EquipScript、UnEquipScriptがそれぞれのタブに表示される  

## ビルドインスクリプトの挿入
アプリ起動時にsample\input\script.ymlが読み込まれる  
読み込み結果はCategoryで折りたたまれ、展開するとビルドインスクリプトのDescが表示名として選択可能になる  


ビルドインスクリプトを選択すると、sample\image\script selected.pngのようにScriptに記載の内容に従って
左側中央、下部の表示が変わる
中央のテキストボックス部分にはArgsで指定されたパラメータが入力できるように、Nameの内容をテキストボックス部分のラベルに表示しされる  
テキストボックス内の検索アイコンは、TypeがValueの場合は無効に、Listの場合は有効になる  
アイコンが有効な状態でクリックするとsample\image\parametor.pngのような画面でListNameで指定されたKey-Valueペアから選択したValueを入力できる  
リストはsample\input\const.ymlに定義され、アプリ起動時に全て読み込まれる  
Key-Valueの選択画面でもフィルタリングは可能で、文字を入力するとsample\image\parametor filtered.pngのようになる

左側下部にはビルドインスクリプトの説明(cript.ymlのDesc)が表示される

ユーザは全てのパラメータを設定後、左右の境界にある->ボタンを押すことで、Script内のScriptに書かれた文字に対して、ARGnをn番目のテキストボックスの入力で置換した文字列が右側のアイテムスクリプトに挿入される
その際に、インデントはアイテムスクリプトの現在のインデント具合に応じて自動で調整すること
